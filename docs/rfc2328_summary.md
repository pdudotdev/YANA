# OSPF Protocol Reference (RFC 2328 — OSPFv2)

## Neighbor State Machine

OSPF routers form adjacencies through a defined state progression:

### States (in order)

- **DOWN**: No Hello packets received from this neighbor. Initial state.
- **INIT**: A Hello was received but the local router's ID was not in the neighbor's Hello. One-way communication only.
- **2WAY**: Bidirectional communication confirmed. Both routers see each other's Router ID in Hello packets. On broadcast networks, DR/BDR election happens here. Non-DR/BDR routers stay in 2WAY (this is normal).
- **EXSTART**: Master/slave negotiation for Database Description (DD) exchange begins. The router with the higher Router ID becomes master.
- **EXCHANGE**: DD packets are exchanged describing each router's LSDB contents.
- **LOADING**: Link State Request packets sent for any LSAs the neighbor has that the local router lacks.
- **FULL**: Databases are fully synchronized. The adjacency is operational.

### Common Stuck States and Causes

| Stuck State | Most Likely Cause |
|-------------|-------------------|
| INIT | One-way: remote side not listing local RID in Hello. ACL blocking 224.0.0.5, auth mismatch on remote, or passive interface on remote. |
| EXSTART/EXCHANGE | MTU mismatch (most common). OSPF includes MTU in DD packets; if received MTU exceeds local interface MTU, DD is rejected. Also: duplicate Router ID causes permanent deadlock. |
| LOADING | Lossy link causing retransmission failures, or premature LSA aging (MaxAge). No timeout defined — stays stuck indefinitely. |

## Hello Protocol

Hello packets maintain neighbor relationships and elect the DR/BDR on broadcast segments.

### Timer Defaults

| Network Type | Hello Interval | Dead Interval |
|-------------|----------------|---------------|
| Broadcast | 10 seconds | 40 seconds |
| Point-to-point | 10 seconds | 40 seconds |
| NBMA | 30 seconds | 120 seconds |
| Point-to-multipoint | 30 seconds | 120 seconds |

Hello and dead timers MUST match on both sides of a link for adjacency to form. A mismatch causes neighbors to never appear or to flap.

### Hello Packet Contents

- Router ID
- Area ID
- Authentication type and data
- Network mask
- Hello interval and dead interval
- DR and BDR IP addresses
- List of neighbors (Router IDs of known neighbors)
- Options field (E-bit for external capability, N-bit for NSSA)

## LSA Types

OSPF uses different LSA types for different routing information:

| Type | Name | Generated By | Flooding Scope | Purpose |
|------|------|-------------|----------------|---------|
| 1 | Router LSA | Every OSPF router | Within area | Describes router's links and costs within an area |
| 2 | Network LSA | DR on broadcast/NBMA segments | Within area | Lists all routers on the broadcast segment |
| 3 | Summary LSA | ABR | Between areas | Advertises inter-area routes (prefixes from other areas) |
| 4 | ASBR Summary | ABR | Other areas | Advertises reachability to an ASBR in another area |
| 5 | AS External LSA | ASBR | Entire AS (except stub areas) | Redistributed routes from other protocols |
| 7 | NSSA External | ASBR inside NSSA | Within NSSA area | Like Type 5 but confined to NSSA; ABR translates to Type 5 |

## Area Types

### Backbone Area (Area 0)
All LSA types allowed. Every area must connect to Area 0 (directly or via virtual link). All inter-area traffic transits Area 0.

### Normal Area
All LSA types allowed. Connects to backbone via an ABR.

### Stub Area
- No Type 5 (external) LSAs allowed
- ABR injects a default route as Type 3 LSA
- Type 3 inter-area summaries are still allowed
- All routers in the area must agree on stub configuration

### Totally Stubby Area
- No Type 3, 4, or 5 LSAs (only Type 1 and 2)
- ABR injects a single default route (Type 3)
- Most restrictive — only intra-area routes plus default
- Cisco proprietary extension; configured with `area X stub no-summary` on ABR

### NSSA (Not-So-Stubby Area)
- No Type 5 LSAs from outside
- Local ASBRs generate Type 7 LSAs (NSSA external)
- ABR translates Type 7 to Type 5 for the backbone
- Default route NOT automatically injected — requires `area X nssa default-information-originate`

### NSSA Totally Stubby
- Combines NSSA + totally stubby: no Type 3, 4, or 5 from outside
- Type 7 generated locally by NSSA ASBR
- ABR injects single default (Type 3)

## DR/BDR Election

On broadcast and NBMA network types, OSPF elects a Designated Router (DR) and Backup DR (BDR):

1. Highest OSPF priority wins (default priority = 1, range 0-255)
2. Priority 0 = ineligible for DR/BDR
3. Ties broken by highest Router ID
4. Election is **non-preemptive**: a higher-priority router joining later does NOT take over DR

All non-DR/BDR routers form FULL adjacency only with DR and BDR. Between themselves they remain in 2WAY state (this is normal, not a problem).

## External Metric Types

Type 5 and Type 7 LSAs carry one of two metric types:

| Metric Type | Cost Calculation | Default |
|-------------|-----------------|---------|
| E1 | External cost + internal OSPF cost to ASBR | - |
| E2 | External cost only (ignores internal distance) | Default on Cisco IOS |

**E1 always beats E2** for the same destination prefix, regardless of metric values. When both exist, E1 wins unconditionally.

## Forwarding Address

A non-zero forwarding address in a Type 5/7 LSA tells routers to forward traffic directly to that IP instead of through the advertising ASBR. If the forwarding address is unreachable via OSPF, the external route is silently discarded (LSA exists in LSDB but route not installed in RIB).

## SPF Algorithm

When the LSDB changes, OSPF runs Dijkstra's Shortest Path First algorithm:
1. Build a tree rooted at the local router using Type 1 (Router) and Type 2 (Network) LSAs
2. Calculate shortest paths to all intra-area destinations
3. Process Type 3 LSAs for inter-area routes
4. Process Type 5/7 LSAs for external routes

SPF is rate-limited by configurable timers (initial delay, hold time, maximum wait) to prevent CPU spikes during rapid topology changes.

## Administrative Distance

OSPF routes have AD 110 by default. This determines preference when multiple routing protocols have routes to the same prefix:

| Protocol | Default AD |
|----------|-----------|
| Connected | 0 |
| Static | 1 |
| eBGP | 20 |
| EIGRP (internal) | 90 |
| OSPF | 110 |
| IS-IS | 115 |
| EIGRP (external) | 170 |
| iBGP | 200 |
